Windows 版 VS Code 将 macOS 的 `Cmd` 替换为 `Ctrl`，`Option` 替换为 `Alt`。以下以 Windows 键位为准。

> 🔄 **从 macOS 迁移？** 几乎所有 `Cmd` 操作在 Windows 上对应 `Ctrl`，`Option` 对应 `Alt`。

## 最核心快捷键（先记住这 10 个）

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+P` | 命令面板（记住这一个就能做所有事） |
| `Ctrl+P` | 快速打开文件 |
| `Ctrl+B` | 切换侧边栏 |
| `Ctrl+J` | 切换面板（终端/输出等） |
| `Ctrl+\` | 分屏 |
| `Ctrl+W` | 关闭当前标签页 |
| `Ctrl+Shift+K` | 删除当前行 |
| `Ctrl+D` | 选中当前词（继续按选中下一个相同词） |
| `Alt+↑/↓` | 向上/下移动当前行 |
| `Ctrl+Shift+Enter` | 在当前行上方插入空行 |

## Windows 特有的注意事项

### 快捷键冲突

以下 VS Code 快捷键可能与 Windows 系统快捷键冲突：

| VS Code 快捷键 | 功能 | Windows 默认用途 |
|---------------|------|-----------------|
| `Alt+F4` | （未使用） | 关闭窗口 |
| `Ctrl+Alt+↑/↓` | 添加上/下行光标 | Intel 显卡旋转屏幕 |
| `Win+.` | 输入表情符号 | Windows Emoji 面板 |

**解决方案：** 卸载冲突的显卡快捷键，或在 VS Code 中自定义快捷键。

### 自定义快捷键

```jsonc
// keybindings.json
[
  // 用 Ctrl+E 替代 Ctrl+P 快速打开
  {
    "key": "ctrl+e",
    "command": "workbench.action.quickOpen"
  },
  // 移除 Ctrl+P 的默认绑定
  {
    "key": "ctrl+p",
    "command": "-workbench.action.quickOpen"
  },
  // 终端切换
  {
    "key": "alt+ctrl+t",
    "command": "workbench.action.terminal.toggleTerminal"
  }
]
```

适用 VS Code 的所有编辑功能与 macOS 版本一致，详见 [macOS 编辑超能力](/docs/macos/04-editing-superpowers)。
